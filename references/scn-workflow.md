# SCN Workflow

Questa reference definisce il processo obbligatorio per costruire una Semantic Content Network.

## Input iniziale

Raccogli sempre:

- topic di riferimento
- brand / progetto / sito
- obiettivo business o query canonica
- audience primaria
- eventuale paese / lingua

Se disponibili, raccogli anche:

- contenuti gia esistenti
- cluster gia coperti
- vincoli editoriali

## Procedura

### Step 1. Definisci l'entita primaria

Identifica il topic, prodotto, concetto o identita che deve diventare il `pillar`.

Verifica:

- rilevanza di business
- potenziale di autorita tematica
- allineamento con cio che il sito vuole essere riconosciuto per

### Step 2. Costruisci i cluster topici

Elenca i sotto-temi che un utente si aspetta di trovare attorno al pillar.

Cerca:

- sotto-domande
- use case
- rischi
- confronti
- meccanismi
- decisioni

### Step 3. Applica i query frame

Per ogni idea, assegna un query frame principale:

- Definition
- Outcome
- Comparison
- Risk
- Use Case
- Mechanism
- Decision

Ogni pagina deve avere un frame dominante. Evita pagine con frame ambiguo o multiplo se non necessario.

### Step 4. Assegna il page type

Usa:

- `Pillar` per la pagina centrale
- `Support` per approfondimenti verticali
- `Bridge` per connettere cluster, audience o topic correlati

### Step 5. Anchor each page in an entity

Per ogni pagina assegna:

- primary entity
- 1-2 supporting entities
- eventuale schema type suggerito

Privilegia entita reali, riconoscibili, categorizzabili.

### Step 6. Verifica distinctiveness e cannibalizzazione

Controlla:

- overlap di intent
- titoli troppo simili
- pagine che rispondono alla stessa domanda con lo stesso angolo

Se trovi overlap:

- cambia query frame
- cambia audience
- cambia angolo
- trasforma una pagina in bridge se serve

### Step 7. Disegna il trust flow

Per ogni pagina definisci il `link intent` prevalente:

- Topical Reinforcement
- Trust Flow Transfer
- Query Disambiguation
- Entity Consolidation
- Navigation Utility

Regole:

- il pillar linka ai support rilevanti
- ogni support linka almeno al pillar
- i support lateralmente collegati possono linkarsi tra loro se condividono contesto o journey

### Step 8. Produci la tabella SCN

La tabella finale deve includere almeno:

- Page Title
- Page Type
- Query Frame
- User Intent
- Primary Entity
- Supporting Entities
- Parent Pillar
- Link Intent
- Suggested Internal Links
- Priority

### Step 9. Produci una pianificazione editoriale

Ordina i contenuti secondo:

- importanza strategica
- autorita del team sul tema
- capacita di generare trust
- dipendenze interne al cluster
- opportunita di linking

La pianificazione deve essere coerente, scalabile e non ridondante.

## Uso di Ahrefs

Se Ahrefs MCP e disponibile:

- usa keyword target e parent topics per rifinire naming, intent e priorita
- non alterare la SCN solo per il volume
- segnala mismatch tra keyword opportunity e semantic fit

## Output minimo richiesto

- 1 pillar
- support pages semanticamente distinte
- eventuali bridge pages
- note di interlinking
- priorita editoriale
- eventuali gap o rischi di overlap
