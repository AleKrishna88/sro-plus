# Gap Analysis

Questa reference definisce come analizzare un corpus di contenuti esistenti e riorganizzarlo in SCN solide.

## Input accettati

- elenco titoli
- elenco URL
- sitemap o export editoriale
- lista di categorie / cluster / tag

## Obiettivi della gap analysis

- raggruppare i contenuti in SCN sensate
- distinguere pillar, support e bridge
- trovare overlap e cannibalizzazione
- individuare contenuti mancanti
- proporre interlinking coerente
- aprire, se richiesto, la fase di scrittura dei contenuti mancanti

## Procedura

### Step 1. Normalizza l'inventario

Per ogni contenuto prova a rilevare:

- titolo
- URL
- topic
- primary entity
- intent dominante
- stage del journey

### Step 2. Clustering semantico

Raggruppa i contenuti per:

- entita primaria
- parent topic
- journey
- query frame

Evita cluster solo tassonomici se non riflettono una vera rete semantica.

### Step 3. Assegna un ruolo SCN

Per ciascun contenuto, valuta:

- `Pillar`
- `Support`
- `Bridge`
- `Out of scope / redundant`

### Step 4. Trova overlap e conflitti

Segnala:

- pagine con stesso intent
- pagine che differiscono solo lessicalmente
- bridge mancanti tra cluster correlati
- support pages isolate senza hub

### Step 5. Identifica i gap

Per ogni cluster, controlla la copertura di:

- Definition
- Outcome
- Comparison
- Risk
- Use Case
- Mechanism
- Decision

Individua:

- frame mancanti
- entita mancanti
- passaggi del journey scoperti
- trust assets mancanti

### Step 6. Progetta l'interlinking

Definisci:

- link da pillar a support
- link di ritorno al pillar
- collegamenti orizzontali tra support correlati
- bridge links tra cluster contigui

Ogni link deve avere un intento esplicito.

### Step 7. Prioritizza

Ordina le lacune da colmare secondo:

- impatto sulla completezza semantica
- rischio di cannibalizzazione
- potenziale di trust flow
- opportunita keyword / SERP, se Ahrefs e disponibile
- fattibilita editoriale

## Uso di Ahrefs

Se Ahrefs MCP e disponibile, usalo per:

- confermare parent topic
- identificare keyword target primarie e secondarie
- leggere gap tra contenuti esistenti e opportunita
- trovare SERP overlap e cannibalizzazione

Se non e disponibile:

- costruisci la gap analysis solo con principi SRO
- suggerisci con chiarezza cosa validare dopo in Ahrefs

## Output minimo richiesto

- mappa delle SCN proposte
- contenuti gia assegnati a ogni SCN
- conflitti / overlap
- contenuti mancanti
- linking strategy
- next steps operativi
