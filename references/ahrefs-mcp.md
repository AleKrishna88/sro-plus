# Ahrefs MCP

Questa reference spiega come usare Ahrefs via MCP senza tradire i principi SRO.

## Principio guida

Ahrefs serve a rifinire e validare.
Non deve decidere da solo architettura, entity map o contenuti.

## Quando usarlo

Usalo per:

- keyword target principale
- keyword secondarie e varianti
- parent topic
- search intent dominante
- difficolta e opportunita
- SERP overlap
- eventuali segnali di cannibalizzazione

## Come integrarlo nella skill

### Per SCN

- usa i dati keyword per affinare naming e priorita
- conferma se un cluster ha domanda reale
- verifica che support e bridge non si pestino i piedi

### Per content development

- scegli una keyword target principale coerente con la pagina
- usa le varianti solo se semanticamente naturali
- evita heading forzati da keyword innaturali

### Per gap analysis

- confronta i contenuti esistenti con opportunita scoperte
- rileva cluster non coperti o coperti male
- trova parent topic o intenti sottorappresentati

## Regole anti-abuso

- non costruire 10 pagine solo per 10 keyword simili
- non sacrificare chiarezza, entity fit o retrieval logic per il volume
- non forzare keyword in H1, opening o heading se il risultato e innaturale
- non usare Ahrefs per giustificare pagine semanticamente duplicate

## Fallback

Se Ahrefs MCP non e disponibile:

- dillo esplicitamente
- continua con entity mapping, query frames e trust flow
- marca keyword target e priorita come `da validare in Ahrefs`
